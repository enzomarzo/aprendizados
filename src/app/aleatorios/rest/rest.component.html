Rest

Ele serve como uma diretriz de como devemos trabalhar com o protocolo HTTP. Nada mais é do que um modelo de arquitetura, um guia que tem como finalidade padronizar e melhorar os envios e recebimentos de dados, independente do formato (Json, xml, textos, etc.)

A ideia é primeiro separar o servidor do cliente, algo muito comum hoje em dia. Na programação web, seria a comunicação entre o back-end (o servidor) e o front-end (cliente).

Essa comunicação entre servidor e cliente é feita com o padrão request-response. Ou seja, fazemos um request para o servidor e recebemos um response dele.

Para fazer uma requisição pelo cliente, com o REST, informamos

  • um verbo: get, post, put, delete..
  • o header: com as informações sobre a requisição (por exemplo se é do tipo xml ou json)
  • o path: o caminho para o servidor (www.site.com.br/api/...)
  • Informações no corpo (body) que são opcionais

Recebendo a requisição o servidor retorna o response com um status, como 200, 201 ou 500.

Soap x Rest x GraphQL

Existe uma diferença teórica entre as três. SOAP é um protocolo (como o protocolo http). REST é um modelo de arquitetura (que usa o protocolo http). E GraphQL é uma linguagem de consultas (query language, como o próprio QL diz).

Todas elas servem para fazer a comunicação entre o backend e o frontend (consumir uma API).

Pensando com cabeça hypada, o SOAP foi substituido pelo REST que está sendo substituido pelo GraphQL. Claro, fora do hype sabemos que não é bem assim e muitas vezes faz sentido usar uma ou outra.

O SOAP é mais verboso, temos que adicionar todos os dados na requisição.

NextJS

O Next é um framework que tem bibliotecas e funcionalidades que facilitam:

  • Criar aplicações que podem ser renderizadas no servidor e gerar sites estáticos
  • Conseguir iniciar um servidor local facilmente
  • Otimização de imagens, script e fontes na hora do build
  • Suporte ao Typescript e ao Sass
  • Rotas: configuradas com a lib Link
  • Internacionalidzação: Criar aplicações em diferentes idiomas
  • Code Splitting and Bundling otimizado
